代理 / IP 配置

配置代理后，服务器重启才能生效（修改 .env 文件中对应字段）。
下方"测试连接"按钮会通过当前已配置的代理检测出口IP，无需重启。
全局代理优先级最高，会覆盖各交易所独立代理。

🌐 全局代理（GLOBAL_PROXY）

同时应用于所有三个交易所。留空则各交易所使用各自配置。

⬡ Decibel 独立代理（DECIBEL_PROXY）

仅用于 Decibel 交易所。支持 HTTP(S) 和 SOCKS5（含用户名密码认证）。

⬡ Extended 独立代理（EXTENDED_PROXY）

仅用于 Extended 交易所。

⬡ RISEx 独立代理（RISEX_PROXY）

仅用于 RISEx 交易所。

⬡ Ondo Perps 独立代理（ONDO_PROXY）

仅用于 Ondo Perps 交易所。

⬡ Perpl 独立代理（PERPL_PROXY）

仅用于 Perpl（perpl.xyz）交易所。

🔍 代理连接测试

验证当前服务器的代理是否正常工作，并显示出口 IP。

⚠ 写入 .env 后需重启服务器才能让代理生效。
不同交易所配置不同代理时，因 Node.js 技术限制，实际生效代理为全局设置（GLOBAL_PROXY）。
如需严格隔离，建议分开部署各交易所版本。
